Track 2 — Development & Local Deployment of Private AI Agents
Ryan Wien · individual entry
An animated AI news anchor that charts stocks, explains the market out loud, takes voice commands and exports reports — with every answer inferred 100% locally on an AMD Radeon GPU via ROCm. Your questions, watchlist and portfolio never leave the machine.

Why private-by-architecture

AI market tools ship your financial curiosity — every question, ticker and position — to a cloud API. Vantage inverts that: the whole agent (reasoning, tool use, memory, report writing) runs against a local open-weights model. One URL flag (?local=1) disables every cloud path.

Agent capabilities

Tool invocation
Charts, watchlist & portfolio ops, navigation, market calendar with on-air alerts — from plain language or voice.
Local multi-turn memory
Last 6 exchanges kept on-device; "what about its risks?" resolves against earlier turns. One-click forget conversation.
Privacy & permission control
Accounts + plan gates; local-only mode; memory clearing; zero external calls in demo data mode.
Report writer
"Write a report and export ppt" → local model writes it, browser builds PowerPoint / Word / Excel. No server.

Experience

Animated anchor reads answers on air (TTS) · voice input · 6 languages · 18 studio sets · live or demo tape.

Inference-speed optimization

  • Q4_K_M quantization — 8B in ~4.9 GB: whole model GPU-resident, no CPU offload split
  • Bounded prompts — memory capped at 6 raw Q/A turns; snapshot is compact JSON → prefill stays constant
  • Streaming — perceived latency ≈ time-to-first-token; the anchor starts speaking mid-decode
  • Right-size switch — one-line swap to a 1B model ≈ halves wall-clock for latency-critical use
